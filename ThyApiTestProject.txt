# QA Engineer - API Automation Case Study (Playwright Edition)

## Görev
TypeScript ve Playwright kullanarak API test otomasyonu projesi oluşturulması bekleniyor. Kodlama standartlarına ve pratiklerine dikkat edilmeli ve kod tekrarı yapılmamalı.

## Beklenenler

### 1. Test Senaryoları 

**User CRUD**
- Kullanıcı listeleme (pagination ile)
- Tekil kullanıcı sorgulama
- Kullanıcı oluşturma (Tüm negative senaryolarla birlikte testler yazılmalı)
- Kullanıcı güncelleme (PUT ve PATCH)
- Kullanıcı silme (DELETE)

**Authentication & Authorization**
- Login flow validasyonu

**Array Response Validasyonu**
- JSON array dönen endpoint'ten (örn: GET /users) belirli bir veriyi çekme ve doğrulama testi

### 2. Test Grupları
Playwright test annotations ile gruplandırma:
- **@smoke:** Kritik API testleri
- **@regression:** Tüm testler

### 3. Design Patterns
Framework'ünüzde design pattern kullanımı beklenmektedir (Singleton, Factory, Builder vb.)

### 4. TestContainers aracılığıyla PostgreSQL Entegrasyonu
TestContainers ile test altyapısında PostgreSQL ayağa kaldırın ve aşağıdaki işlevleri implement edin:
- API request ve response'ları database'e kaydetme

**Database Connection:**
property dosyasından configürasyon bilgilerini okusun. Fakat pipeline'a dönük olarak ortam değişkeniyle ezilebilsin. (DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD)

**Not:** Test projesi koşulurken schema yoksa otomatik olarak yaratsın, tablo yoksa yaratsın varsa kullansın. Schema tasarımı size kalmış.
